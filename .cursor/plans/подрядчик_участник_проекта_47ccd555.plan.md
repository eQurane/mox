# План: участник-подрядчик и видимость по типу задания

## Исходное состояние

- В [`server/src/routes/projects.js`](c:\Users\eQurane\VSCode\mox\server\src\routes\projects.js) для **Внешний подрядчик** на `GET /api/projects` и `GET /api/projects/:id` сразу **`403`**; в **`ASSIGNABLE_ROLE_NAMES`** этой роли нет — добавить в проект как участника нельзя.
- Задачи уже хранят тип исполнителя в **`tasks.role_id`** (см. **`TASK_ROLE_NAMES`** в [`server/src/routes/tasks.js`](c:\Users\eQurane\VSCode\mox\server\src\routes\tasks.js)). Это и есть «исполнитель задания — внешний подрядчик».

## Правило видимости (принято по вашему выбору)

Учётная запись подрядчика видит контент **только если одновременно**:

1. есть активное членство **`user_project`** (`excluded_at IS NULL`);
2. связанное **Задания** имеет **`tasks.role_id`** с именем роли **`Внешний подрядчик`** (join на `roles.name`).

Списки проектов — как у исполнителя: **только проекты из `user_project`** (без фильтра по наличию задание подрядчика).
