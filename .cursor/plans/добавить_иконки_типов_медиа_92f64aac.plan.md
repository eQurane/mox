## Текущее состояние

На странице **`#/media`** уже реализовано корректное отображение медиа-превью через функцию `attachMediaCardThumb()`:
- Если файл — изображение → показать картинку с placeholder, при ошибке загрузки → иконка типа
- Если не изображение → сразу иконка типа (video, audio, table, docs)
- Размер иконок в CSS: **56×56 px** (исходные SVG **24×24**, масштабируются CSS)

Но на страницах **`#/home`** (список проектов), **`#/tasks`**, **`#/collections`** и **на деталях проекта/ТЗ/коллекции** карточки показывают только `project-card__media--placeholder` без медиа.

## Требуемые изменения

### 1. Проверить данные из API и при необходимости расширить их

- `server/src/routes/collections.js` — добавить поле со статусом или последним медиа коллекции (если его нет)

### 3. Расширить `mediaCardThumb.js` для отображения `no-image` иконки

При отсутствии медиа нужно показывать иконку `no-image-24.svg`:
- Добавить экспортированную функцию `renderNoImageIcon()` или расширить `renderKindIcon()`
- При вызове `attachMediaCardThumb()` с пустым медиа-объектом → показать `no-image`
